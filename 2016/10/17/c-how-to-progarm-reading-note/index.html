运算符重载

  1. 编写 非static成员 函数或者 全局函数 就可以实现运算符的重载。不能是static成员函数，因为运算符必须作用在对象上，而不是类

  2. 运算符重载不改变元数，优先级和结合律

  3. 运算符重载函数的参数要求至少有一个是自己定义的类对象，不能全是基本类型

  4. 注意 () [] -> 必须作为类的成员函数进行重载

  5. 来看看重载 << 和 >> 的示例代码：

  6. C++通过 new 和 delete 可以在运行时动态的分配和释放内存，这种内存是在 自由存储区 ，也称之为 堆(heap) 。例如 ： TIme *timePtr = new Time; 他会分配一个Time对象大小的内存空间，调用Time类的构造函数，并将该内存空间的首地址给timePtr指针；要释放这个内存空间，调用 delete timePtr; 即可，他会先调用Time的析构函数，再释放内存

  7. 还可以使用 new 和 delete 来动态的分配和释放数组内存。

继承

  1. 继承方式有3种，public protected 和 private .这里的继承方式和数据成员的不是一个意思
  2. 继承之后，基类的private数据成员也是被继承下来的，但是派生类不能直接访问
  3. 友元函数不会被继承
  4. 尽量避免在基类中使用protected成员，而应该使用private成员
  5. 派生类不会继承基类的构造函数、析构函数和重载的赋值运算符
  6. 先创建的对象后被析构

多态性

  1. 多态性的两个底层技术：virtual函数和动态绑定

  2. 如果没有运用virtual函数的话，将派生类的地址赋给基类指针，调用同名函数，执行的还是基类中的函数，而不是派生类中重写的函数。因为调用函数取决于调用函数的句柄（指针）的类型，而不是句柄指向的对象的类型。这样的话，并不能实现多态。

  3. 但是使用virtual函数的话，调用哪个函数，就取决与句柄指向的对象。也就是在基类中使用virtual定义同名函数，在派生类中重写该函数

  4. 一旦一个函数被声明为virtual函数，那么在继承层次中，所有下面的这个函数也都是virtual函数

  5. 通过以下两种形式，可以动态的执行函数。这些函数是在运行时绑定上去的

    1. 通过指向派生类的基类指针调用virtual函数： shapePtr->draw();
    2. 通过指向派生类的基类引用调用virtual函数： shapeRef.draw();

  6. 通过按名引用特定对象和使用点成员运算符调用virtual函数，是静态绑定，也就是在编译时就绑定了，不存在多态。 squareObject.draw();

  7. 将类中的一个或多个函数声明为纯玩virtual函数，可以使一个类成为抽象类。纯虚函数不需要提供函数的实现

  8. 实例化一个抽象类会导致编译错误
