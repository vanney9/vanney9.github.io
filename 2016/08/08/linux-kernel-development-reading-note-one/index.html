Linux内核设计与实现 读书笔记（一）

笔记

Linux内核简介

  1. Unix中，所有的东西都被当做文件对待。这种抽象使对数据和对设备的操作都是通过一套相同的系统调用接口来进行的。
  2. 操作系统：在整个系统中负责完成最基本功能和系统管理的部分。这部分包括内核、设备驱动程序、启动引导程序、命令行Shell等等。其中 内核 是操作系统的核心。
  3. 内核独立于普通程序，它一般处于系统态，拥有受保护的内存空间和访问硬件的所有权限。这种系统态加上被保护的内存空间，称为内核空间。相对的，一般的应用程序在 用户空间 执行。
  4. 应用程序可以通过 系统调用 来与内核通信。也就是应用程序调用库函数（API）来让内核代其完成一些任务。
  5. 处理器的任何时间都处在下面3者之一：
    1. 运行于用户空间，执行用户进程。
    2. 运行于内核空间，处于进程上下文，代表某个特定的进程执行。
    3. 运行于内核空间，处于中断上下文，与任何进程无关，处理某个特定的中断。、
  6. 比较内核的两大类型
    1. 单内核：把内核作为一个单独的大过程来实现，同时也运行在一个单独的地址空间上面。因此，这样的内核通常以单个静态二进制文件的形式存放于磁盘中。内核间通信简单。这种形式的内核简单，性能高。
    2. 微内核：微内核的功能被划分为多个独立的过程，保持独立并运行在各自的地址空间。使用 进程间通信机制（IPC） 来联系各个过程。模块化，好管理。
  7. Linux是单内核，但是它也实现了模块化等微内核的优点。

进程管理

  1. 进程：进程就是处于执行器的程序。但是进程不仅仅是一段可执行代码，它还包括其他资源：打开的文件，挂起的信号，内核内部数据，处理器状态，内存地址以及一个或多个执行线程。所以说， 一个进程包括一个或者多个线程。

  2. 线程：执行线程，简称线程，是 进程中活动的对象。内核调度的是线程，不是进程。

  3. 进程提供两种虚拟机制：

    1. 虚拟处理器：让进程觉得自己独享处理器，其实是多个进程共享处理器
    2. 虚拟内存：让进程觉得自己拥有整个系统的内存资源。

  4. 注意进程并不是程序，它是处于 执行期程序 和 相关资源 的总称。可以同时存在多个进程，他们是执行的同一个程序。

  5. 父进程调用系统函数fork();生成子进程。接着子进程调用exec();这组函数皆可以创建新的地址空间，将新的程序载入其中，开始一段新的旅程。exit();函数会退出程序执行，终结进程释放资源。父进程还可以使用wait4();函数等待子进程的结束。

  6. 内核将进程的列表放在 任务队列 的双向循环列表中。列表中的每一项都是类型为 task_struct的结构体，称为 进程描述符 ；它包含了一个进程的具体信息：比如进程打开的文件，进程的地址空间，挂起的信号等等信息。

  7. 内核通过唯一的进程标识符 PID (process identification value) 来标识每一个进程。PID是一个数，int类型。每个进程的PID存放在他们的task_struct结构体中。

  8. 可以通过current宏查出当前正在执行的进程。需要经常的访问进程描述符。

  9. 进程描述符的state成员描述了进程的当前状态。进程必然处于以下5种状态之一：

    1. TASK_RUNNING：运行态。进程是可执行的。他说明进程正在运行或者在运行队列中等待执行。
    2. TASK_INTERRUPTIBLE：可中断。进程正在睡眠，也就是被阻塞，等待某条件的达成。一旦条件达成，就会将状态设置为运行态。
    3. TASK_UNINTERRUPTIBLE：不可中断。在睡眠，但是不会被唤醒。。。使用的较少
    4. __TASK_TRACED：被其他进程跟踪的进程
    5. __TASK_STOPPED：停止态。进程停止执行。

  10. set_task_state(task, state);函数用来设置进程的状态。

  11. 可执行程序代码：将一个可执行文件载入到进程的地址空间执行。一般的进程在用户空间执行，当一个进程执行了 系统调用 或者是 触发了某个异常 时，他会 陷入 内核空间。此时称 内核代表进程执行，并处于进程上下文 。

  12. 有一个 进程树 。所有的进程都是PID为1的 init进程 的后代。进程描述符中有指向父进程的parent指针和指向子进程的children 子进程链表。

  1. 进程创建：分成两个步骤 fork() 和 exec()

  2. fork() ：由父进程调用。通过拷贝当前进程来创建一个子进程。父进程和子进程的区别仅仅是 PID 和 PPID (parent PID) ，以及一些资源和统计量

    1. exec() ：由子进程调用。负责读取可执行文件，并将它载入到该进程的地址空间，进行执行。

  3. fork()使用 写时拷贝 (copy-on-write) 。简单说就是需要拷贝时才去拷贝，高效。

  4. 线程机制提供了在同一程序内共享内存地址空间的一组线程。这组线程还共享打开的文件和其他资源。线程机制支持并发设计。

  5. Linux中并没有线程这一特殊的定义。它只是把线程当做一个和其他进程共享地址空间以及其他资源的特殊进程而已。

  6. 进程结束时会调用exit();函数释放持有的内存空间等资源。一般在main函数返回时，都会隐式的调用exit();函数。

进程调度

  1. 调度程序负责决定将哪个进程投入运行，何时运行以及运行多长时间。调度程序是像Linux这样的多任务操作系统的基础。
  2. 在一组处于可执行状态的进程中，选择一个来执行，是调度程序所需完成的基本工作。
  3. 多任务 ：指的是能同时并发的交互执行多个进程。它能使多个进程在内存中，但是只有一个处于可执行的状态。
  4. 多任务系统分成2类：非抢占式和抢占式。抢占式也就是调度系统可以强制让一个进程挂起，执行另一个进程。所以在进程被挂起之前执行的时间称之为 进程的时间片 。非抢占式，除非进程自己停止执行，否则会一直运行下去。
  5. 进程调度算法其实就是用来决定这个时候哪个进程来执行，执行多久而已。有很多种进程调度算法。
  6. 进程优先级：优先级高的进程被优待。
  7. 时间片：表示进程一次能持续运行的时间。时间片太长，感觉不是并发的，交互性不好；太短，又会使太多的性能浪费在进程的切换上面。
