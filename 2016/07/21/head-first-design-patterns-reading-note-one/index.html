简介

  1. 设计原则1：找出应用中可能需要变化的地方，把他独立出来。
  2. 设计原则2：针对接口编程，而不是针对实现编程。多态。
  3. JAVA接口？
  4. 设计原则3：多用组合，少用继承。
  5. 策略模式：定义了算法族，分别封装起来，让他们之间可以互相替换，此模式让算法的变化独立于使用算法的客户。todo add picture

观察者模式

  1. todo add picture
  2. 观察者模式提供了一种对象设计，让主题和观察者之间松耦合。他们可以交互，但是不太清楚彼此的细节。主题只知道观察者实现了一个Observe的接口，主题也只需要一个实现observe接口的类的list。可以随时添加/删除观察者，而不改动主题的代码。观察者需要只需要知道主题就行。
  3. 设计原则4：为了交互对象之间的松耦合设计而努力。松耦合就是要将对象之间的互相依赖降到最低。
  4. 观察者模式可以让主题主动去推送消息，也可以让观察者自己去拉取主题的数据。

装饰者模式

  1. 设计原则5：类应该对扩展开放，对修改关闭。允许类容易扩展，在不修改现有代码的情况下，就可搭配新的行为。
  2. 装饰者模式：动态的将责任附加到对象上，若要扩展功能，装饰者提供了比继承更有弹性的替代方案。todo add 3 picture
  3. 装饰者和被装饰者有着共同的超类。装饰者可以有自己的额外的功能实现。

工厂方法模式

  1. 把创建对象的部分提取出来，给一个新对象。这个对象只负责创建对象。

  2. 工厂方法用来处理对象的创建，但是具体的创建行为却是封装在子类中。所以工厂方法一般都是抽象的。

  3. 来通过一张图认识工厂方法模式：todo add a picture

  4. 工厂方法模式：定义了一个创建对象的接口，但由子类决定要实例化的类是哪一个。工厂方法只是一个抽象的，他将类的实例化推迟到子类中。注意工厂方法还可以创建一系列的抽象类，不只是一个类。

单例模式

  1. 确保一个类只有一个实例，并提供一个全局访问点。
  2. 注意多线程可能会导致生成多个实例，要去避免。
